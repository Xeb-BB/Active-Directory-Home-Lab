# Lab Module: Group Policy Management
## Introduction
### Centralized management is the primary reason organizations leverage Active Directory. This lab module demonstrates how to enforce system configurations, implement security baselines, and restrict user capabilities at scale using Group Policy Objects (GPOs). By applying these rules to specific Organizational Units (OUs), you will replicate how professional IT departments lock down corporate environments.

## Tools & Technologies Used
### Group Policy Management Console (gpmc.msc)

### Active Directory Users and Computers (dsa.msc)

### Group Policy Management Editor (Targeting Administrative Templates)

### Operating Systems: Windows Server 2022 and Windows 10 Client (Logged in as a domain user)

### Command Line Interface: gpupdate /force for policy propagation

## What to Expect in This Section
### Best practices for structuring Organizational Units (OUs) to isolate target users and computers from the rest of the domain.

### A workflow for creating unique, descriptive Group Policy Objects (ex. Restrict_Control_Panel).

### Step-by-step navigation through administrative paths to modify default OS behaviors.

### Linking mechanisms that tie policies directly to your custom OUs, followed by instant client-side testing and enforcement.
