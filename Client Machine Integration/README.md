# Lab Module: Client Machine Integration
## Introduction
### An Active Directory environment is only functional when client workstations are successfully connected to it. This lab module guides you through the process of bridging the gap between infrastructure and endpoints. You will configure a standalone Windows client workstation, align its networking properties with your server, and securely join it to the central corporate domain to simulate a live enterprise fleet.

## Tools & Technologies Used
### Windows 10 Pro (Client Workstation)

### Windows Server 2022 (Active Directory Domain Controller / css.com)

### Windows Advanced System Settings (Domain Join Utility)

### IPv4 Properties (Manual DNS pointing setup)

### Command Line Interface: ipconfig /all and system properties verification

## What to Expect in This Section
### Instructions on transitioning a workstation from a standard local workgroup into a centralized domain.

### Steps to correctly point client network adapter properties to the primary Domain Controller's IP.

### A clear sequence for executing the domain join process using authenticated Active Directory administrative credentials.

### Post-join validation steps to ensure the client machine is visible within the Active Directory computer container.
